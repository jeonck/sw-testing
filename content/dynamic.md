# 동적 테스트 기법 (Dynamic Testing Techniques)

## 블랙박스 기법

블랙박스 테스트는 내부 구조를 알지 못한 채 입력과 출력만을 기반으로 테스트를 수행하는 기법입니다. 이는 사용자 관점에서 시스템을 검증하는 데 중점을 둡니다.

### 1. 동등 분할 (Equivalence Partitioning)

#### 정의
- 입력 데이터를 동등한 그룹으로 분할하여 각 그룹에서 하나의 값을 선택하여 테스트
- 동일한 그룹 내의 값은 동일한 방식으로 처리된다고 가정

#### 적용 방법
- **유효 클래스**: 시스템이 정상적으로 처리하는 입력 값 범위
- **무효 클래스**: 시스템이 정상적으로 처리하지 않는 입력 값 범위

#### 예시
```
ID 입력 필드: 3~15자리의 영문자와 숫자만 허용
- 유효 클래스: [3, 15] 범위의 영문자와 숫자 조합
- 무효 클래스: 2자리 이하, 16자리 이상, 특수문자 포함
```

#### 장점
- 테스트 케이스 수 감소
- 효율적인 테스트
- 중요한 결함 탐지

### 2. 경계값 분석 (Boundary Value Analysis)

#### 정의
- 입력값의 경계 범위를 중심으로 테스트
- 대부분의 결함이 경계 근처에서 발생한다는 가정

#### 적용 방법
- **기본 경계값**: 최소값, 최대값, 최소값-1, 최대값+1, 유효 범위 내 임의 값
- **이론적 경계값**: 정의된 범위의 경계

#### 예시
```
나이 입력 필드: 1~100세
- 테스트 값: 0, 1, 2, 99, 100, 101
```

#### 고급 경계값 분석
- 다변수 경계 분석
- 출력 경계 분석
- 내부 경계 분석

### 3. 결정 테이블 (Decision Table)

#### 정의
- 다양한 조건 조합에 대한 동작을 테이블 형식으로 표현
- 복잡한 비즈니스 로직 검증에 적합

#### 구성 요소
- **조건 항목**: 시스템에 영향을 주는 조건
- **동작 항목**: 조건에 따라 수행되는 동작
- **규칙**: 조건과 동작의 조합

#### 예시
| 규칙 | 로그인 상태 | 결제 금액 > 100,000 | 할인 적용 | 배송비 면제 |
|------|-------------|---------------------|-----------|-------------|
| 1    | 예          | 예                  | 예        | 예          |
| 2    | 예          | 아니오              | 아니오    | 아니오      |
| 3    | 아니오      | -                   | 아니오    | 아니오      |

#### 장점
- 복잡한 조건 논리 표현
- 누락된 조건 식별
- 테스트 커버리지 향상

### 4. 상태 전이 (State Transition)

#### 정의
- 시스템의 상태 변화와 이에 따른 동작을 테스트
- 상태 다이어그램 기반 테스트

#### 구성 요소
- **상태**: 시스템이 가질 수 있는 조건
- **이벤트**: 상태 변경을 유발하는 트리거
- **전이**: 상태 간의 변화
- **행동**: 전이 시 수행되는 동작

#### 예시
```
은행 계좌 상태:
- 상태: 잔액 부족, 정상, 정지
- 이벤트: 입금, 출금, 정지 요청
- 전이: 잔액 부족 → 정상 (입금 시)
```

#### 상태 전이 다이어그램
```
[로그인 전] --(로그인)---> [로그인 상태] --(로그아웃)---> [로그인 전]
     |                           |                           |
     |----(로그인 실패)--------->|                           |
```

### 5. 유스케이스 테스트

#### 정의
- 사용자의 시나리오를 기반으로 테스트 케이스 작성
- 실제 사용자 관점에서 시스템 테스트

#### 구성 요소
- **액터**: 시스템과 상호작용하는 사용자 또는 시스템
- **유스케이스**: 시스템이 제공하는 기능
- **시나리오**: 특정 상황에서의 유스케이스 실행

#### 테스트 설계
1. 유스케이스 다이어그램 작성
2. 주요 시나리오 식별
3. 예외 시나리오 식별
4. 테스트 케이스 작성

## 화이트박스 기법

화이트박스 테스트는 내부 구조와 로직을 기반으로 테스트를 수행하는 기법입니다. 코드 커버리지를 중심으로 테스트를 설계합니다.

### 1. 구문(Statement) 커버리지

#### 정의
- 프로그램 내 모든 문장(Statement)을 최소한 한 번은 실행하는 테스트

#### 계산식
```
구문 커버리지 = (실행된 문장 수 / 전체 문장 수) × 100%
```

#### 예시
```java
public int calculate(int a, int b, String operation) {
    int result = 0;  // 문장 1
    if (operation.equals("add")) {  // 문장 2
        result = a + b;  // 문장 3
    } else if (operation.equals("sub")) {  // 문장 4
        result = a - b;  // 문장 5
    }
    return result;  // 문장 6
}
```
- 테스트 1: calculate(5, 3, "add") → 문장 1, 2, 3, 6 실행
- 테스트 2: calculate(5, 3, "sub") → 문장 1, 2, 4, 5, 6 실행
- 결과: 100% 구문 커버리지

### 2. 결정(Decision) 커버리지

#### 정의
- 모든 분기(Decision)에 대해 참/거짓 두 가지 결과를 최소한 한 번씩 테스트

#### 계산식
```
결정 커버리지 = (실행된 분기 수 / 전체 분기 수) × 100%
```

#### 예시
```java
if (a > 0) {  // 분기 1
    result = a * 2;
} else {
    result = a * -1;
}

if (b < 10) {  // 분기 2
    result = result + 1;
}
```
- 테스트 1: a = 5, b = 5 → 분기 1(true), 분기 2(true)
- 테스트 2: a = -5, b = 15 → 분기 1(false), 분기 2(false)
- 결과: 100% 결정 커버리지

### 3. 조건 커버리지

#### 정의
- 모든 조건이 참/거짓 결과를 최소한 한 번씩 갖도록 테스트

#### 예시
```java
if (a > 0 && b < 10) {  // 조건 A: a > 0, 조건 B: b < 10
    result = a + b;
}
```
- 테스트 1: a = 5, b = 5 → A: true, B: true
- 테스트 2: a = -5, b = 15 → A: false, B: false
- 결과: 100% 조건 커버리지

### 4. MC/DC 커버리지 (Modified Condition/Decision Coverage)

#### 정의
- 각 조건이 결과에 독립적으로 영향을 미치는 것을 증명하는 테스트
- 항공/의료 등 안전 중요 시스템에 사용

#### 요구사항
1. 모든 결과 값이 최소 한 번씩 발생
2. 모든 조건이 최소 한 번씩 결과에 영향을 미침
3. 각 조건이 다른 조건의 영향 없이 결과에 영향을 미침

## 경험 기반 기법

경험 기반 테스트는 테스터의 경험, 직관, 지식을 기반으로 테스트를 설계하는 기법입니다.

### 1. 오류 추정 (Error Guessing)

#### 정의
- 과거의 경험과 직관을 바탕으로 오류가 발생할 가능성이 높은 영역을 추정

#### 적용 방법
- 과거 프로젝트의 결함 분석
- 테스터의 경험
- 유사 시스템 분석
- 테스트 설계 문서 검토

#### 일반적인 오류 패턴
- 공백 또는 NULL 값 처리
- 0 또는 음수 처리
- 배열 경계
- 동시성 문제
- 리소스 누수

### 2. 탐색적 테스트 (Exploratory Testing)

#### 정의
- 테스트 설계와 실행을 동시에 수행하는 테스트 기법
- 테스터의 창의성과 직관 활용

#### 특징
- 사전 테스트 설계 없음
- 즉석에서 테스트 설계
- 실시간 피드백 반영
- 창의적인 테스트 수행

#### 수행 단계
1. 테스트 목적 설정
2. 테스트 설계 및 실행
3. 결과 분석
4. 다음 테스트 계획

#### 장점
- 창의적 테스트 가능
- 예상치 못한 결함 발견
- 시간 절약
- 테스터 능력 활용

#### 단점
- 재현 어려움
- 문서화 부족
- 일관성 부족
- 테스터 의존도 높음

## 테스트 기법 선택 전략

### 1. 테스트 목적에 따른 선택

| 테스트 목적 | 추천 기법 |
|-------------|-----------|
| 기능 검증 | 블랙박스 기법 |
| 로직 검증 | 화이트박스 기법 |
| 경험 활용 | 경험 기반 기법 |
| 결함 예방 | 정적 테스트 |

### 2. 테스트 수준에 따른 적용

| 테스트 레벨 | 주요 적용 기법 |
|-------------|----------------|
| 단위 테스트 | 화이트박스 기법 |
| 통합 테스트 | 블랙박스 + 화이트박스 |
| 시스템 테스트 | 블랙박스 기법 |
| 인수 테스트 | 블랙박스 기법, 유스케이스 테스트 |

### 3. 커버리지 목표 설정

- **단위 테스트**: 80% 이상 결정 커버리지
- **통합 테스트**: 70% 이상 경로 커버리지
- **시스템 테스트**: 95% 이상 요구사항 커버리지

이러한 동적 테스트 기법들은 서로 보완적으로 사용되어야 하며, 프로젝트의 특성과 위험 요소에 따라 적절한 기법을 선택하고 조합하는 것이 중요합니다.
