# 저사양 기기 및 열악한 네트워크(3G/LTE) 환경 강제 설정 테스트

테스트 환경이 항상 쾌적한 사무실 Wi-Fi와 최신 플래그십 스마트폰일 수는 없습니다. 실제 사용자는 지하철, 엘리베이터, 해외 오지에서 구형 단말기로 우리 서비스를 이용할 수 있습니다.

시스템의 **회복력(Resilience)**과 안정성을 검증하기 위한 '악조건 속의 테스팅' 실무 가이드를 정리해 드립니다.

## 📂 [테스트 수행] 극한 환경 테스트: 저사양 기기 및 열악한 네트워크

### 1. 열악한 네트워크 환경 강제 설정 (Network Throttling)

네트워크가 느려지거나 끊길 때 앱이 무한 로딩에 빠지거나 데이터가 꼬이는 현상을 잡아내야 합니다.

#### ① PC 웹: 크롬 개발자 도구 활용

- **설정 방법**: Network 탭 > No Throttling 드롭다운 클릭 > Slow 3G 선택 또는 Add...를 통해 초당 전송량(Custom)을 극도로 낮게 설정.

- **체크포인트**:
  - 로딩 바(Spinner)가 적절히 노출되는가?
  - 타임아웃 발생 시 사용자에게 명확한 에러 메시지를 보여주는가?
  - 데이터 로딩 도중 페이지를 이탈했을 때 앱이 크래시(Crash) 나지 않는가?

#### ② 모바일: 개발자 모드 및 프록시 도구 활용

- **iOS**: 설정 > 개발자(Developer) > Network Link Conditioner 메뉴에서 'Very Bad Network' 등을 선택.

- **Android**: 별도의 앱(예: Netcut)이나 Charles Proxy 같은 도구의 'Throttle' 기능을 사용하여 PC와 연결된 모바일의 속도를 제어.

### 2. 저사양 기기(Low-End Device) 테스트 전략

메모리(RAM)와 CPU 성능이 낮은 기기에서는 평소에 안 보이던 자원 고갈 관련 버그가 튀어나옵니다.

- **메모리 압박 테스트**: 여러 앱을 동시에 실행한 상태에서 테스트를 진행하거나, 설정의 '백그라운드 프로세스 제한'을 1개로 설정하여 테스트.

- **저장공간 부족 테스트**: 기기 용량을 99%까지 채운 뒤 앱을 실행하거나 파일 다운로드/업로드를 시도.

- **체크포인트**:
  - 고화질 이미지가 노출되는 화면에서 앱이 버벅이거나 튕기지 않는가?
  - 백그라운드로 갔다가 돌아왔을 때 데이터가 초기화되지 않는가? (메모리 부족으로 인한 앱 재실행 확인)

### 3. 전환 환경 테스트 (The Switch Test)

단순히 '느린 것'보다 더 무서운 것은 **'상태가 변하는 순간'**입니다.

- **Wi-Fi ↔ LTE 전환**: 사무실 밖으로 나가거나 Wi-Fi를 끄는 순간, 스트리밍이나 결제 프로세스가 끊기지 않고 재연결(Retry)되는지 확인.

- **Airplane Mode 활용**: 결제 전송 버튼을 누른 직후 비행기 모드를 켜서 '네트워크 연결 끊김' 시나리오 강제 수행. (데이터 정합성 확인)

### 4. 실무 체크리스트 (극한 환경용)

| 점검 항목 | 테스트 액션 | 기대 결과 |
|-----------|-------------|------------|
| **타임아웃** | Slow 3G 환경에서 대용량 데이터 조회 | 일정 시간 후 "네트워크가 원활하지 않습니다" 안내 노출 |
| **중복 클릭** | 속도가 느린 상태에서 전송 버튼 연타 | 첫 번째 요청만 처리되고 이후 클릭은 무시됨 |
| **이미지 처리** | 저사양 기기에서 이미지 리스트 스크롤 | 이미지 로딩 전 플레이스홀더(Placeholder) 노출 및 매끄러운 스크롤 |
| **비정상 종료** | 데이터 전송 중 강제로 네트워크 차단 | 앱이 중단되지 않고 로컬에 임시 저장하거나 재시도 안내 |

### 5. 실무 보고서 작성 팁

#### [결함 요약] 
네트워크 지연 환경에서 결제 완료 버튼 중복 클릭 시 이중 결제 발생

#### [테스트 환경] 
Network: Charles Proxy를 이용한 Throttle 설정 (Latency: 2000ms, Speed: 56kbps)
Device: Samsung Galaxy A10 (저사양 단말기)

#### [문제점] 
서버 응답이 오기 전까지 버튼이 활성화되어 있어 사용자가 여러 번 클릭 가능함. 클라이언트단에서 클릭 방지(Debounce) 처리가 미흡함.

## 💡 테스터를 위한 한 줄 전략

"사용자의 환경은 항상 최악일 수 있다고 가정하세요." 최신 아이폰으로만 테스트하는 것은 반쪽짜리 테스트입니다. 네트워크를 죽이고, 메모리를 괴롭힐 때 비로소 서비스의 진정한 '품질'이 드러납니다.
