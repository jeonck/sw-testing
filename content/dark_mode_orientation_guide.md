# 다큽모드, 가로/세로 전환, 앱 백그라운드 진입 시 발생하는 흔한 결함들

모바일 앱 테스팅에서 기능 구현만큼 중요한 것이 **'시스템 상태 변화'**에 따른 안정성입니다. 사용자가 화면을 돌리거나, 잠시 카톡을 보러 나가는 등의 일상적인 동작이 앱에는 치명적인 오류를 일으키는 원인이 되곤 합니다.

실무에서 가장 빈번하게 발생하는 3대 환경 변화 결함과 체크포인트를 정리해 드립니다.

## 📂 [테스트 수행] 시스템 상태 변화에 따른 주요 결함 리포트

### 1. 다큽모드(Dark Mode) 전환: "사라진 글자와 배경"

단순히 색상만 바뀌는 것이 아니라, 리소스가 다시 로드되는 과정에서 UI 결함이 집중됩니다.

#### 흔한 결함:

- **텍스트 소실**: 배경은 검은색으로 변했는데, 글자색이 여전히 검은색(Hard-coded)이라 글자가 안 보이는 현상.

- **아이콘 가시성**: 어두운 배경에서 어두운 톤의 아이콘이 묻혀버리는 문제.

- **일관성 결핍**: 일부 팝업이나 웹뷰(WebView)만 라이트모드 색상으로 남아 있어 눈을 피로하게 만드는 현상.

#### 체크포인트: 
시스템 설정에서 모드를 바꾼 직후, 현재 열려 있는 화면의 색상이 즉시 반영되는가? (앱 재시작 없이 확인)

### 2. 가로/세로 회전(Orientation): "잘려나간 버튼과 초기화"

화면 비율이 변할 때 레이아웃이 깨지거나, 내부 데이터가 날아가는 현상이 발생합니다.

#### 흔한 결함:

- **레이아웃 붕괴**: 가로 모드에서 버튼이 화면 밖으로 밀려나거나, 키보드가 올라올 때 입력창을 가려버리는 현상.

- **데이터 초기화**: 화면이 회전되면서 액티비티(Activity)가 재시작되어, 입력 중이던 텍스트나 스크롤 위치가 처음으로 돌아가는 현상.

- **이미지 왜곡**: 이미지 비율(Aspect Ratio)이 유지되지 않고 옆으로 길게 늘어지는 문제.

#### 체크포인트: 
회원가입이나 긴 설문조사 입력 중 화면을 돌렸을 때, 입력한 내용이 그대로 유지되는가?

### 3. 백그라운드 진입 및 복귀: "죽어버린 세션과 좀비 프로세스"

홈 버튼을 눌러 앱을 내렸다가 다시 올릴 때(Foreground) 발생하는 결함들입니다.

#### 흔한 결함:

- **세션 만료 및 튕김**: 백그라운드에 있는 동안 보안 토큰이 만료되어 복귀 시 에러 팝업과 함께 앱이 종료되는 현상.

- **미디어 컨트롤 오류**: 유튜브나 음악 앱처럼 백그라운드에서 소리가 계속 나야 하는데 멈추거나, 반대로 멈춰야 하는데 계속 나오는 현상.

- **화면 멈춤(Freeze)**: 복귀 직후 데이터 동기화 과정에서 화면이 몇 초간 굳어버리는 현상.

#### 체크포인트: 
앱을 백그라운드에 둔 상태에서 다른 무거운 앱(카메라, 고사양 게임)을 실행한 뒤 복귀해도 앱이 정상적으로 살아있는가?

## 📝 테스터를 위한 상황별 실무 체크리스트

| 구분 | 주요 테스트 액션 | 성공 기준 (Expected Result) |
|------|------------------|---------------------------|
| **다크모드** | 설정창에서 모드 변경 후 앱 복귀 | 모든 텍스트와 아이콘이 배경색과 대비되어 명확히 보임 |
| **화면 회전** | 동영상 재생 중 화면 가로 전환 | 재생 위치가 끊기지 않고 전체 화면으로 부드럽게 전환됨 |
| **백그라운드** | 결제 요청 버튼 클릭 직후 홈 버튼 클릭 | 앱 복귀 시 결제가 정상 처리되거나, 안전하게 취소 안내됨 |
| **복합 상황** | 가로 모드 상태에서 백그라운드 진입 후 복귀 | 화면 방향과 데이터 상태가 진입 전과 동일하게 유지됨 |

## 💡 실무자 팁: "인터럽트(Interrupt) 테스트"

위의 세 가지 상황을 합쳐서 테스트해 보세요. 예를 들어 "가로 모드에서 데이터를 입력하다가(회전), 전화가 와서 앱이 내려가고(백그라운드), 전화를 끊고 다시 복귀했을 때" 앱이 멀쩡하다면 그 앱의 품질은 아주 훌륭하다고 볼 수 있습니다.
