# [기법] 영리한 테스트 설계

**"무식하게 다 누르지 마세요. 이론을 섞으세요."**

## 상세 내용

### 경계값 분석 활용

- **입력 유효성 검증**: 최소/최대값 근처에서 예외 발생 여부 확인
- **성능 경계**: 처리 가능한 최대 데이터량 확인
- **시간 경계**: 타임아웃, 스케줄링 관련 테스트

### 동등 분할의 전략

- **유효 클래스**: 시스템이 정상 처리할 수 있는 입력 범위
- **무효 클래스**: 시스템이 정상 처리하지 않을 입력 범위
- **대표값 선정**: 각 클래스에서 의미 있는 대표값 선택

### 실무 팁 - 블랙박스 테스트

- **사용자 시나리오 기반**: 실제 사용 패턴을 반영한 테스트 케이스 작성
- **우선순위 기반 설계**: 핵심 기능에 더 많은 테스트 리소스 투입
- **데이터 중심 접근**: 다양한 입력 조합을 고려한 테스트 데이터 설계

### 사례 - 블랙박스 테스트

> 전자상거래 사이트에서 할인율 계산 로직에 대해 경계값 분석을 적용해, 100% 할인 조건이 발생하는 치명적 결함을 사전에 발견.
