# [리포팅] 버그 전달의 기술

**"잘 쓴 리포트 하나가 열 번의 대화보다 낫습니다."**

## 상세 내용

### 결함 보고서 구성

- **재현 절차**: 단계별로 명확하게 기술 (예: 1. 로그인 → 2. 메뉴 이동 → 3. 기능 클릭)
- **기대 결과 vs 실제 결과**: 명확한 비교
- **스크린샷/로그**: 시각적 증거 첨부
- **심각도/우선순위**: 적절한 수준 설정

### 커뮤니케이션 전략

- **사일 기반 보고**: 감정 표현이 아닌 사실 중심
- **재현 가능성**: 동일 조건에서 반복 가능한지 명시
- **비즈니스 영향**: 사용자에게 미치는 영향 설명

### 실무 팁 - 결함 관리

- **5단계 재현법**: 단순한 단계로 결함 재현이 가능하도록
- **환경 정보 명시**: OS, 브라우저, 버전 등 명확히 기술
- **일관성 유지**: 동일한 유형의 결함은 동일한 형식으로 보고

### 사례 - 결함 관리

> 결함 보고서에 "버튼이 안 눌립니다" 대신 "iOS 15.4 환경에서 결제 버튼 클릭 시 30% 확률로 반응 없음. 로그에 에러 발생"으로 기술하여 개발자 해결 시간 60% 단축.
