# 결정 테이블 상세 설명

## 결정 테이블 작성 4단계

### 1단계: 조건(Conditions)과 결과(Actions) 식별

먼저 시스템의 입력 조건과 그에 따른 출력 행동을 모두 나열합니다.

- **조건**: 테스트에 영향을 주는 입력 변수 (예: 회원 여부, 쿠폰 보유 여부)
- **결과**: 조건에 따라 발생하는 시스템 반응 (예: 10% 할인, 배송비 무료)

### 2단계: 규칙(Rules)의 개수 계산

모든 조건이 **True(Y) / False(N)**로 나뉜다면, 가능한 규칙의 총 개수는 $2^n$ ($n$ = 조건의 개수)입니다.

- 조건이 3개라면? $2^3 = 8$개의 규칙(세로 열)이 생깁니다.

### 3단계: 테이블 구성 (조합 채우기)

조건의 조합을 빠짐없이 기입합니다. 이때 규칙적으로 채우면 실수를 줄일 수 있습니다.

- 첫 번째 조건: Y 4개, N 4개
- 두 번째 조건: Y 2개, N 2개, Y 2개, N 2개
- 세 번째 조건: Y, N, Y, N... 반복

### 4단계: 결과(Actions) 기입 및 최적화

각 조건 조합에 맞는 결과를 체크($\text{X}$)합니다. 만약 특정 조건이 결과에 영향을 주지 않는다면 해당 칸을 대시($-$)로 표시하고 규칙을 합칠 수 있습니다.

## 📝 실습 예시: 쇼핑몰 할인 혜택

**조건**: (1) 회원인가? (2) 5만원 이상 구매했는가?  
**결과**: (A) 10% 할인 적용 (B) 무료 배송

| 구분 | 항목 | 규칙 1 | 규칙 2 | 규칙 3 | 규칙 4 |
|------|------|--------|--------|--------|--------|
| 조건 | 회원 여부 | Y | Y | N | N |
|      | 5만원 이상 | Y | N | Y | N |
| 결과 | 10% 할인 | X |   |   |   |
|      | 무료 배송 | X | X |   |   |
