# 자주 묻는 질문 (FAQ) - 소프트웨어 테스팅

## Q1. 테스팅이란 무엇인가요?

**A:** 테스팅은 소프트웨어가 요구사항을 충족하는지 확인하고 결함을 식별하기 위한 체계적인 과정입니다. 이는 소프트웨어 품질 보장의 핵심 활동으로, 사용자에게 신뢰성 있는 제품을 제공하는 데 중요한 역할을 합니다.

## Q2. 테스트의 주요 목표는 무엇인가요?

**A:** 테스트의 주요 목표는 다음과 같습니다:
- 결함 탐지 및 제거
- 품질 보증
- 위험 감소
- 요구사항 검증
- 시스템 신뢰성 향상

## Q3. 블랙박스 테스트와 화이트박스 테스트의 차이점은 무엇인가요?

**A:** 
- **블랙박스 테스트**: 내부 구조를 알지 못한 채 입력과 출력만을 기반으로 테스트하는 방식입니다. 주로 기능 테스트에 사용됩니다.
- **화이트박스 테스트**: 내부 구조와 로직을 기반으로 테스트하는 방식입니다. 코드 커버리지 등을 검증할 때 사용됩니다.

## Q4. 결정 테이블(Decision Table)이란 무엇인가요?

**A:** 결정 테이블은 다양한 조건 조합에 따른 시스템의 동작을 표 형식으로 표현한 테스트 기법입니다. 조건(Conditions)과 결과(Actions)로 구성되어 복잡한 비즈니스 로직을 시각적으로 표현할 수 있습니다. 자세한 내용은 [결정 테이블 상세 설명](#decision_table)을 참조하세요.

## Q5. 결정 테이블 작성 4단계는 무엇인가요?

**A:** 결정 테이블 작성은 다음과 같은 4단계로 구성됩니다:

1. **조건(Conditions)과 결과(Actions) 식별**: 시스템의 입력 조건과 그에 따른 출력 행동을 모두 나열합니다.
2. **규칙(Rules)의 개수 계산**: 가능한 규칙의 총 개수는 $2^n$ ($n$ = 조건의 개수)입니다.
3. **테이블 구성 (조합 채우기)**: 조건의 조합을 빠짐없이 기입합니다.
4. **결과(Actions) 기입 및 최적화**: 각 조건 조합에 맞는 결과를 체크하고 규칙을 합칠 수 있습니다.

자세한 내용은 [결정 테이블 상세 설명](#decision_table)을 참조하세요.

## Q6. 테스트 자동화를 언제 도입해야 하나요?

**A:** 테스트 자동화는 다음과 같은 경우에 도입하는 것이 좋습니다:
- 반복적인 회귀 테스트가 필요한 경우
- 수동 테스트가 시간이 많이 소요되는 경우
- 테스트 커버리지를 높이고 싶은 경우
- CI/CD 파이프라인에 통합하고자 할 때

## Q7. 단위 테스트, 통합 테스트, 시스템 테스트의 차이점은 무엇인가요?

**A:** 
- **단위 테스트**: 개별 구성 요소나 함수 수준에서의 테스트 (예: 개별 함수, 클래스)
- **통합 테스트**: 여러 구성 요소 간 상호작용을 검증하는 테스트
- **시스템 테스트**: 전체 시스템의 기능 및 비기능 요구사항을 검증하는 테스트
