# 핫픽스(Hotfix) 상황에서 최소한으로 챙겨야 할 스모크 테스트(Smoke Test)

운영 환경에서 긴급하게 결함을 수정하는 핫픽스(Hotfix) 상황은 테스터에게 가장 압박이 심한 순간입니다. 시간이 극도로 부족하기 때문에 모든 기능을 볼 수 없으며, 이때 수행하는 것이 바로 **스모크 테스트(Smoke Test)**입니다.

"불을 지폈을 때 연기가 나는지(기초적인 동작 여부)만 확인한다"는 의미처럼, 시스템의 가장 핵심적인 혈관이 제대로 도는지 검증하는 최소한의 가이드를 정리해 드립니다.

## 📂 [테스트 수행] 핫픽스 상황의 스모크 테스트 전략

### 1. 스모크 테스트의 철학: "얕고 넓게"

핫픽스 스모크 테스트는 특정 기능을 깊게 파는 것이 아니라, 서비스 전체가 '동작 불능' 상태가 아닌지를 빠르게 훑는 것이 목적입니다.

- **수행 시간**: 전체 서비스 규모에 따라 15분~30분 내외로 종료.

- **통과 기준**: 주요 화면 진입 가능 여부, 핵심 버튼 작동 여부.

### 2. 핫픽스 시 반드시 포함해야 할 '5대 핵심 체크리스트'

#### ① 크리티컬 패스(Critical Path) 점검

서비스의 존재 이유가 되는 핵심 동선을 확인합니다.

- **쇼핑몰**: 로그인 → 상품 검색 → 장바구니 담기 → 결제 완료.

- **커뮤니티**: 로그인 → 게시글 작성 → 댓글 작성 → 삭제.

- **금융**: 계좌 조회 → 이체 비밀번호 입력 → 이체 완료.

#### ② 수정 지점의 직접 영향도 확인

핫픽스로 고친 바로 그 부분이 제대로 작동하는지 확인합니다.

- **예**: "결제 버튼이 안 눌려요"라는 핫픽스였다면, 결제 버튼 클릭 후 결제창이 정상적으로 뜨는지 확인.

#### ③ 주요 페이지 진입 및 UI 깨짐 확인

수정 과정에서 공통 CSS나 JS가 망가져 전체 레이아웃이 깨지는 경우가 빈번합니다.

- **체크**: 메인 홈, 마이페이지, 상품 상세 등 주요 페이지 3~5곳을 진입해보고 화면이 정상적으로 렌더링되는지 확인.

#### ④ 로그인 및 세션 유지 확인

의외로 핫픽스 배포 후 세션 처리가 잘못되어 사용자가 튕기거나 로그인이 안 되는 사고가 많습니다.

- **체크**: 로그인 후 페이지를 이동해도 로그인이 유지되는지, 로그아웃은 정상적으로 되는지 확인.

#### ⑤ 환경별 확인 (최소 2곳)

시간이 없다면 가장 사용자가 많은 환경 두 곳만 봅니다.

- **예**: PC(Chrome) + Mobile(가장 많이 쓰는 기기 1종).

### 3. 핫픽스 스모크 테스트 수행 시나리오 예시

| 단계 | 테스트 항목 | 확인 사항 (Pass/Fail) |
|------|-------------|----------------------|
| 1 | 앱 실행/웹 접속 | 첫 화면이 3초 이내에 정상 노출되는가? |
| 2 | 로그인 | 기존 계정으로 로그인이 성공하는가? |
| 3 | 수정 사항 확인 | 이번 핫픽스의 원인이 된 결함이 수정되었는가? |
| 4 | 핵심 액션 | 결제, 예약, 전송 등 가장 중요한 버튼이 작동하는가? |
| 5 | 로그아웃/종료 | 앱이 비정상 종료(Crash) 없이 안전하게 닫히는가? |

### 4. 실무 팁: "무엇을 버릴 것인가?"

핫픽스 상황에서는 다음 항목들을 과감히 생략하거나 사후 확인으로 돌립니다.

- **하위 호환성**: 오래된 OS 버전이나 비주류 브라우저 테스트.

- **상세 UI**: 오타, 약간의 정렬 어긋남, 색상 차이 등.

- **예외 케이스**: 말도 안 되는 데이터 입력, 복잡한 할인 조건 조합 테스트 등.

## 📝 핫픽스 승인 보고 양식 (간결함이 핵심)

#### [Hotfix 스모크 테스트 결과 - 승인]

- **수정 결함**: 주문서 내 쿠폰 적용 오류 수정 확인 (Pass)

- **스모크 테스트 범위**: 로그인 > 상품 상세 > 주문서 진입 > 결제 완료 (총 15분 소요)

- **결과**: 핵심 프로세스 이상 없음. 안드로이드 최신 기기 1종 점검 완료.

- **특이사항**: 레이아웃 미세 정렬 불일치 1건 발견했으나 서비스 이용에 지장 없어 배포 후 수정 권고.

## 💡 테스터를 위한 한 줄 전략

"핫픽스 상황에서 테스터는 완벽주의자가 아닌, 소방관이 되어야 합니다." 집 전체를 수리하기보다, 지금 당장 번지고 있는 불을 끄고 퇴로(핵심 기능)를 확보하는 것이 최우선입니다.
