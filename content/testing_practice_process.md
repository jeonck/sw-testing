# [프로세스] 실무 생존 가이드

**"출근부터 배포까지, 테스터의 일과는 이렇습니다."**

## 상세 내용

### 테스트 계획

- **범위 정의**: 무엇을 테스트할지 명확히 정의
- **리스크 분석**: 핵심 기능, 고위험 모듈 식별
- **일정 수립**: 개발 일정과 연계한 테스트 일정 계획
- **자원 배정**: 테스터 역할 및 책임 정의

### 테스트 설계

- **요구사항 기반 테스트**: 명세서 기반 테스트 케이스 설계
- **경험 기반 테스트**: 테스터 경험을 활용한 테스트 설계
- **도구 활용**: 테스트 관리 도구(TestRail, Zephyr 등) 활용

### 테스트 실행

- **자동화 vs 수동**: 적절한 테스트 유형 선택
- **회귀 테스트 전략**: 변경 영향 분석 및 테스트 범위 결정
- **결함 추적**: 결함 수명주기 관리

### 실무 팁 - 테스트 프로세스

- **일일 스크럼 활용**: 개발자와의 커뮤니케이션 강화
- **테스트 커버리지 모니터링**: 테스트 진행 상황 시각화
- **지속적 피드백**: 품질 상태에 대한 지속적 보고

### 사례 - 테스트 프로세스

> 애자일 팀에서 스프린트 중간에 테스트 시작으로 결함 수정 주기 단축. 테스트 자동화 도입으로 회귀 테스트 시간 70% 절약.
